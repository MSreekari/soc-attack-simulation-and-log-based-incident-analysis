# SSH - Brute Force Attack SImulation and Log Analysis 

**SSH -** 
* It stands for Secure Shell and runs on **port 22.**
* It is a network protocol to gain remote access of a system.
* It encrypts the data between the client and the server thus preventing Man-In-The-Middle(MITM) attacks.
* It is used to access a computer miles away open terminal, execute commands, move files, and share files.
* It is preferred over protols like telnet (Port 23) because in telnet, the data is not encrypted, the data can be clearly read.
